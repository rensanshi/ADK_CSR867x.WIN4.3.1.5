Detailed Description

If the application has enabled caller id notifications at the AG, this message will be sent to the application whenever a caller id notification is received from the AG. Note: These messages can own a 2nd heap cell through the member: caller_name. Thus they should never be delete using the MessageCancelXXX API's as this will lead to a memory leak.
